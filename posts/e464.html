linux中chmod与chown两个命令详解

发表于 2020-02-26 更新于 2020-08-30 分类于 linux

如果创建了一个目录或文件，需要改变它的安全性设置，在linux系统中有chmod和chown命令来完成这项任务。但两个命令的用途是不同的，chmod命令用来改变文件和目录的权限，而chown命令用来改变文件的属主和属组。

chmod

命令格式：chmod options mode file
options：
-v：显示权限变更的详细资料
-R：将权限的改变递归作用到文件和子目录。

文件和目录的权限分为3组，文件属主的权限，属组成员的权限，其他用户的权限。

[ ugoa …] [[+ - =] [rwxXstugo]…]
第一组字符定义了权限作用的对象：

  • u代表用户
  • g代表组
  • o代表其他
  • a代表以上所有

第二个符号：

  • 表示增加权限 　　 - 表示取消权限　　= 表示唯一设定权限。

第三个符号代表作用到设置上的权限：

  • r 表示可读取
  • w 表示可写入
  • x 表示可执行
  • X 表示只有当该文件是个子目录或者该文件已经被设定过为可执行，赋予执行权限。
  • s：运行时重新设置UID或GID。
  • t：保留文件或目录。
  • u：将权限设置为跟属主一样。
  • g：将权限设置为跟属组一样。
  • o：将权限设置为跟其他用户一样。
    最常用的比如：chmod -R 777 Downloads/*
    设置文件夹下所有文件可读可写。

chown

命令格式：chown options owner[.group] file
只有root有使用权限，改变文件的所属关系。
注意：只有root用户能够改变文件的属主。任何属主都可以改变文件的属组，但前提是属主必须是原属组和目标属组的成员。
options：
-v：显示指令执行过程
-R：将权限的改变递归作用到文件和子目录。
-h：改变该文件的所有符号链接的所属关系。
-f：不显示错误信息

owner：新的文件拥有者的属主ID
group：新的文件的属组

比如：chown -R wry:web /tmp/wry命令是将/tmp/wry这个目录的属主改为wry，属组为web。

权限：r- -　可读，八进制值4
　　：-w-　可写，八进制值2
　　：- -x　可执行，八进制值1
